## Bit Period: The Foundation of UART Timing

The baud rate tells both devices how long each bit lasts. The math is simple:

```
Bit Period (seconds) = 1 / Baud Rate
```

At 115200 baud:

```
Bit Period = 1 / 115200 = 8.68 µs
```

Every bit holds its value for exactly that long. The receiver samples the line at the *middle* of each bit period — this gives the most margin for timing drift.

## How the Receiver Locks On

Without a shared clock, the receiver must determine *when* each bit starts. It does this by detecting the falling edge of the start bit:

```
                    ┌─ sample here (middle of bit period)
                    │
IDLE  START  D0  D1 ▼ D2  ...
  1     0    ?   ?  ?  ?
        │
        └─ falling edge: receiver starts its internal timer
```

After detecting the start bit, the receiver waits **1.5 bit periods** to sample D0, then exactly **1 bit period** for each subsequent data bit.

## Frame Format: 8N1

The most common UART frame format is **8N1**:

- **8** — 8 data bits
- **N** — No parity bit
- **1** — 1 stop bit

This is almost universally the default. When someone says "UART at 115200", they mean 115200 8N1.

## Parity: Optional Error Detection

UART supports an optional parity bit inserted between the last data bit and the stop bit:

| Parity | Meaning |
|--------|---------|
| None (N) | No parity bit — most common |
| Even (E) | Bit set so total 1s in frame is even |
| Odd (O)  | Bit set so total 1s in frame is odd |

```c
// Setting parity in STM32 HAL
huart2.Init.Parity = UART_PARITY_NONE;  // 8N1
// huart2.Init.Parity = UART_PARITY_EVEN; // 8E1
```

Parity catches single-bit errors but not burst errors. For reliable comms, use a proper CRC over the entire packet instead.

## Stop Bits: 1 or 2?

Most systems use 1 stop bit. Two stop bits exist for legacy compatibility with older hardware that needed more time to process between frames. You'll essentially never need 2 stop bits on modern microcontrollers.

## Clock Accuracy Requirements

Because UART is asynchronous, the transmitter and receiver clocks must stay synchronized across a full frame (10 bits for 8N1). The cumulative drift must not exceed ±0.5 bit periods.

In practice, the clocks must agree to within about **±2–3%**. Most crystal oscillators are accurate to ±50 ppm (0.005%), so timing errors are rarely a problem unless you're using an RC oscillator.

```c
// RC oscillators can drift ±1-5% — always calibrate if using for UART
// On STM32, HSI oscillator is ±1% at room temperature
```

## Calculating Max Baud Rate

The maximum baud rate is limited by:

1. **GPIO slew rate** — the pin's max switching speed
2. **Peripheral clock** — the UART peripheral needs at least 16× oversampling
3. **Cable capacitance** — longer cables limit speed

On an STM32F4 running at 84 MHz:

```
Max UART clock = 84 MHz / 16 = 5.25 Mbaud
```

Practically, 1–3 Mbaud is reliable on short PCB traces.
